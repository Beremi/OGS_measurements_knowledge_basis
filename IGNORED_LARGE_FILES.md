## Generated Measurement Mirror

`cda_knowledge_base/measurements_info/` is a generated navigation mirror. It
copies each curated measurement folder, builds `MEASUREMENT_INFO.md`, and carries
generated inventories/extracts such as:

- `measurement_info_manifest.csv`
- `deep_source_index.md`
- `deep_source_index.csv`
- `measurement_content_deep_dive.md`
- `measurement_content_deep_dive.csv`
- `workbook_sheet_deep_index.csv`
- per-stream `MEASUREMENT_INFO.md`

Regenerate from the repository root after the measurement source-file copies are
available:

```bash
python3 cda_knowledge_base/scripts/build_measurement_deep_source_index.py
python3 cda_knowledge_base/scripts/build_measurement_content_deep_dive.py
python3 cda_knowledge_base/scripts/build_measurements_info_mirror.py
```

The first two commands rebuild the source indexes and mined content summaries
under `cda_knowledge_base/measurements/`. The final command rebuilds the ignored
`cda_knowledge_base/measurements_info/` mirror. These scripts depend on the local
measurement source files and Python packages used by the repo, including
spreadsheet/PDF readers.

## Measurement Source-File Copies

`cda_knowledge_base/measurements/**/source_files/` contains curated local copies of
raw or near-raw files: workbooks, PDFs, presentations, ZIPs, layout files, and
provider-response notes. These copies are omitted to keep GitHub small.

They are not generated from the tracked docs alone. To restore them after a fresh
clone, rehydrate from the raw local source locations and transfers described in the
catalogue, primarily:

- `cda_knowledge_base/attachments/`
- `cda_knowledge_base/attachments/thunderbird_recovered/`
- `cda_knowledge_base/file_transfers/collected/`
- external TeamBeam/Gmail/source-transfer backups when the local raw file is not
  present in the clone

After restoring the source-file copies, rerun the measurement mirror commands
above to refresh generated indexes.

## Generated OGS/Inversion Runs

`SOTA_OGS_Mont_Terri_work/inversion_workflow/runs/` contains generated candidate
run directories. A typical run directory may include copied OGS XML/VTU inputs,
patched output variables, generated `bulk_w_projections.vtu`, OGS output VTUs,
sampled state tables, objective/evaluation CSVs, and audit JSON/Markdown files.

Full regeneration requires the model inputs, processed observation tables, and
either an OGS executable or an OGS container image. From
`SOTA_OGS_Mont_Terri_work/`, rebuild the model-facing observation layers if they
are stale:

```bash
python3 inversion_workflow/scripts/build_processed_observations.py
python3 inversion_workflow/scripts/build_mesh_observation_lookup.py
python3 inversion_workflow/scripts/build_permeability_observation_targets.py
python3 inversion_workflow/scripts/build_state_observation_targets.py
```

Then regenerate a single run:

```bash
python3 inversion_workflow/scripts/evaluate_inversion_candidate.py \
  --config inversion_workflow/run_config.example.json \
  --run-id <run-id> \
  --ogs-mode execute \
  --ogs /path/to/ogs \
  --overwrite
```

or with the transferred Apptainer/Singularity image restored outside Git:

```bash
python3 inversion_workflow/scripts/evaluate_inversion_candidate.py \
  --config inversion_workflow/run_config.example.json \
  --run-id <run-id> \
  --ogs-mode execute \
  --sif /path/to/apptainer_ogs6.5.4.sif \
  --overwrite
```

Use `--ogs-mode dry-run` only to record the command/status without producing full
OGS output VTUs. To evaluate a table of candidates, use:

```bash
python3 inversion_workflow/scripts/run_inversion_candidate_search.py \
  --candidate-table <candidate-table.csv> \
  --config inversion_workflow/run_config.example.json \
  --ogs-mode execute \
  --ogs /path/to/ogs
```

The generated `ogs_state_samples.csv` files are written by
`sample_ogs_state_outputs.py`. Diagnostic ERT residual tables named
`ert_resistivity_diagnostic.csv` are generated separately after OGS outputs exist:

```bash
python3 inversion_workflow/scripts/evaluate_ert_resistivity_diagnostic.py \
  --ogs-output-dir inversion_workflow/runs/<run-id>/ogs_output \
  --support-mesh inversion_workflow/runs/<run-id>/bulk_w_projections.vtu \
  --output inversion_workflow/runs/<run-id>/ert_resistivity_diagnostic.csv \
  --timestep-output inversion_workflow/runs/<run-id>/ert_resistivity_diagnostic_timesteps.csv \
  --summary-output inversion_workflow/runs/<run-id>/ert_resistivity_diagnostic_summary.json \
  --markdown-output inversion_workflow/runs/<run-id>/ert_resistivity_diagnostic.md
```

## Listing Or Refreshing This Manifest

List exact currently ignored files:

```bash
git ls-files --others --ignored --exclude-standard
```

Check why a path is ignored:

```bash
git check-ignore -v <path>
```

Refresh the ignored-size summary:

```bash
python3 - <<'PY'
import os, subprocess
from collections import defaultdict
from pathlib import Path

paths = subprocess.check_output(
    ["git", "ls-files", "--others", "--ignored", "--exclude-standard", "-z"],
    text=False,
).decode("utf-8", "replace").split("\0")
paths = [p for p in paths if p and os.path.isfile(p)]

def human(n):
    units = ["B", "KiB", "MiB", "GiB"]
    x = float(n)
    for unit in units:
        if x < 1024 or unit == units[-1]:
            return f"{x:.1f} {unit}" if unit != "B" else f"{int(x)} B"
        x /= 1024

groups = defaultdict(lambda: [0, 0])
for p in paths:
    if p.startswith("SOTA_OGS_Mont_Terri_work/inversion_workflow/runs/"):
        group = "generated OGS/inversion runs/"
    elif p.startswith("cda_knowledge_base/measurements_info/"):
        group = "generated measurement mirror/"
    elif p.startswith("cda_knowledge_base/measurements/") and "/source_files/" in p:
        group = "measurement source-file copies/"
    elif "__pycache__/" in p or p.endswith(".pyc"):
        group = "Python cache artifacts"
    else:
        group = "other ignored patterns"
    groups[group][0] += os.path.getsize(p)
    groups[group][1] += 1

print(f"Total ignored: {human(sum(os.path.getsize(p) for p in paths))} / {len(paths)} files")
for group, (size, count) in sorted(groups.items(), key=lambda item: item[1][0], reverse=True):
    print(f"{human(size):>10} {count:6d} {group}")
PY
```
